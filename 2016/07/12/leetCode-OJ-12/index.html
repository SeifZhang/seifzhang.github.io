Algorithm Description

Given an integer, convert it to a roman numeral.

Input is guaranteed to be within the range from 1 to 3999.

相同问题

My Algorithm

idea:

【罗马数字】

1~9: {“I”, “II”, “III”, “IV”, “V”, “VI”, “VII”, “VIII”, “IX”};

10~90: {“X”, “XX”, “XXX”, “XL”, “L”, “LX”, “LXX”, “LXXX”, “XC”};

100~900: {“C”, “CC”, “CCC”, “CD”, “D”, “DC”, “DCC”, “DCCC”, “CM”};

1000~3000: {“M”, “MM”, “MMM”}.
例如：3999—MMMCMXCIX——128—CXXVIII
罗马数字规则：

1， 罗马数字共有7个，即I（1）、V（5）、X（10）、L（50）、C（100）、D（500）和M（1000）。

罗马数字中没有“0”。

2， 重复次数：一个罗马数字最多重复3次。

3， 右加左减：

在较大的罗马数字的右边记上较小的罗马数字，表示大数字加小数字。

在较大的罗马数字的左边记上较小的罗马数字，表示大数字减小数字。

4， 左减的数字有限制，仅限于I、X、C，且放在大数的左边只能用一个。

(*) V 和 X 左边的小数字只能用Ⅰ。

(*) L 和 C 左边的小数字只能用X。

(*) D 和 M 左 边的小数字只能用C。

code:

public class Solution {
 public String intToRoman(int num) {
        String[][] RomanDict = new String[][] {
                { "", "I", "II", "III", "IV", "V", "VI", "VII", "VIII", "IX" },
                { "", "X", "XX", "XXX", "XL", "L", "LX", "LXX", "LXXX", "XC" },
                { "", "C", "CC", "CCC", "CD", "D", "DC", "DCC", "DCCC", "CM" },
                { "", "M", "MM", "MMM", "", "", "", "", "", "", "" }, };
        // 3，2，1，0行分别处理千，百，十，个位
            return RomanDict[3][num / 1000] + 
            RomanDict[2][num % 1000 / 100] + 
            RomanDict[1][num % 100 / 10]+ 
            RomanDict[0][num % 10];
    }
}
